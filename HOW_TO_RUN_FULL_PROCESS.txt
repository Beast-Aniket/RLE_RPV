
1) START
- python -m venv .venv
- source .venv/bin/activate
- pip install -r requirements.txt
- streamlit run app.py

2) LOGIN
- Predefined CCF only:
  username: BEAST
  password: admin123

3) CCF WORKFLOW
A. User Management tab
   - create CLERK/ADMIN/FINAL users
   - edit users
   - disable users

B. Session/Exam Upload tab
   - create session (only session name)
   - choose faculty
   - choose exam + program code from dropdown or create new
   - upload students (.csv/.xlsx/.dbf)

C. Audit tab
   - view all activity logs
   - download audit CSV

4) CLERK WORKFLOW
- login with CCF-created clerk user
- select session + exam
- search student by PRN/Seat
- enter missing GPIs
- if all GPIs available and remark has RLE, it is auto-removed
- for RPV, clerk removes remark manually and submits

5) ADMIN WORKFLOW
- login with CCF-created admin user
- view all or pending requests
- filters: date, session, exam, PRN, seat
- today's report / yesterday's report quick buttons
- approve/reject/suggest
- on approve, PDF is generated automatically
- admin can bulk download PDFs by filters (ZIP)

6) FINAL MEMBER WORKFLOW
- login with CCF-created FINAL user
- filter by session/exam/day
- download filtered entries CSV
- select multiple requests and mark DONE/PENDING
- PDF visibility available from approved entries

7) FILES GENERATED
- university_rle_rpv.db
- university_rle_rpv_schema.sql
- university_rle_rpv_dump.sql
- generated_letters/*.pdf

8) IMPORT STANDARD CUSTOMIZATION
- edit `import_config.py`
- change aliases in COLUMN_ALIASES to match your file structure
RLE-RPV UNIVERSITY PORTAL - FULL PROCESS RUN GUIDE

1) INSTALL + START
------------------
1. Open terminal in project folder.
2. Run:
   python -m venv .venv
   source .venv/bin/activate
   pip install -r requirements.txt
   python app.py
3. Open browser: http://localhost:5000

2) DATABASE FILES (AUTO CREATED)
--------------------------------
After app start, these files are created in current folder:
- rle_rpv.db          -> main runtime SQLite DB
- rle_rpv_schema.sql  -> schema SQL definition
- rle_rpv_dump.sql    -> SQL dump (updated after every create/update action)

3) LOGIN DETAILS
----------------
CCF: ccf / ccf123
Clerk users (faculty wise):
- clerk_snt / clerk123
- clerk_com / clerk123
- clerk_inter / clerk123
- clerk_hum / clerk123
Admin users (faculty wise):
- admin_snt / admin123

- admin_hum / admin123
Final member: final_member / final123

4) COMPLETE END-TO-END FLOW
---------------------------
STEP A - CCF FLOW
A1. Login as CCF.
A2. Create exam session (example: April 2025).
A3. Upload faculty-wise student result data (.csv/.xlsx/.dbf).
A4. Verify upload summary and audit logs.
A5. Optional: Download SQL dump from CCF dashboard.

STEP B - CLERK FLOW
B1. Login with faculty clerk account.
B2. Select session and search student by PRN or seat number.
B3. For RLE:
    - add missing sem GPI fields
    - system recalculates CGPI in request payload
B4. For RPV:
    - check "If RPV, set Eligible" option
    - status changes to ELIGIBLE and RPV removed from remark
B5. Submit request to admin.
B6. Track requests in tabs:
    - Pending
    - Approved
    - Rejected
    - Suggested Edit

STEP C - ADMIN FLOW
C1. Login with faculty admin account.
C2. Open Pending requests.
C3. Review payload and decide:
    - Approve  -> student data updated + letter generated
    - Reject   -> moves to Rejected (comment required)
    - Suggest  -> moves to Suggested Edit (comment required)
C4. Use status tabs for tracking approved/rejected/suggested records.

STEP D - FINAL MEMBER FLOW
D1. Login as final_member.
D2. View date-wise + session-wise summary with pending counts.
D3. Download:
    - single letter
    - selected letters ZIP
    - day-wise CSV report
D4. Update final state per record:
    - PENDING
    - DONE
    - QUERY (with comment)

5) SAMPLE DATA
--------------
Use sample_students.csv to test quick upload.

6) TROUBLESHOOTING
------------------
- If login fails: ensure app started first time so users are seeded.
- If .xlsx upload fails: verify openpyxl installed from requirements.
- If DB lock issue occurs: stop duplicate running app processes.
